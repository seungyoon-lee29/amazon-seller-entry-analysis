# 데이터 분석 포트폴리오 프로젝트 추천

> 작성일: 2026-06-12 · 대상: 이커머스/핀테크 데이터 분석 직무 지원
> 전제: 대표 프로젝트 1개를 깊이 있게. 아마존 셀러 실데이터는 없음(경험만 보유), 투자 대회는 수상 이력만 보유.

---

## 1. 핵심 전략: "경험을 데이터로 재해석하는 사람"

신입/주니어 데이터 분석가 포트폴리오의 90%는 **남의 데이터로 남의 질문에 답하는** 구조입니다 (Olist, Titanic, 신용카드 사기 탐지 등). 채용 담당자가 가장 지겨워하는 유형이기도 합니다.

당신의 차별화 포인트는 반대입니다:

- **아마존 셀러로서 실제로 내렸던 의사결정**(소싱, 가격, 광고, 재고)이 있고
- 그 의사결정의 맥락·제약·실패 경험을 **도메인 지식으로 보유**하고 있으며
- 이제 그 의사결정을 **데이터로 검증하고 시스템화**할 수 있는 기술을 배웠다는 것

즉 포트폴리오의 서사를 "데이터셋을 분석해봤다"가 아니라 **"내가 사업할 때 감으로 했던 판단을, 지금이라면 데이터로 이렇게 했을 것이다"** 로 잡는 것이 핵심입니다. 이 서사는 면접에서 어떤 꼬리 질문이 와도 본인 경험으로 답할 수 있어 방어력이 압도적입니다. 이커머스 회사 지원 시 도메인 적합성도 자동으로 증명됩니다.

---

## 2. 대표 프로젝트 (1순위 추천)

### "아마존 신규 셀러 카테고리 진입 의사결정 분석"
**— 전직 셀러가 데이터로 다시 풀어보는 상품 소싱·시장 기회 탐색**

#### 한 줄 컨셉
"아마존 셀러 시절 나는 어떤 카테고리에 어떤 가격대로 진입할지 감과 블로그 정보로 결정했다. 이 프로젝트는 그 의사결정 문제를 공개 데이터로 정량화한다: **어떤 카테고리/니치가 신규 셀러에게 진입 가치가 있는가?**"

#### 왜 이 주제인가
- 시장 기회 분석(market opportunity analysis)은 이커머스 기업의 전략기획·카테고리 매니지먼트 부서가 실제로 하는 일과 동일 → 데이터 분석 + 전략 기획 둘 다 어필 가능
- 리뷰 EDA·구매자 세그먼트 분석은 흔하지만, **셀러(공급자) 관점의 분석은 드묾** → "다들 하는 뻔한 것"을 피하면서도 검증된 데이터셋 사용 가능
- Jungle Scout, Helium 10 같은 셀러 분석 SaaS가 파는 기능을 직접 구현하는 셈 → 셀러 경험자만 낼 수 있는 문제 정의

#### 데이터
| 소스 | 내용 | 비고 |
|---|---|---|
| [Amazon Reviews 2023 (McAuley Lab)](https://huggingface.co/datasets/McAuley-Lab/Amazon-Reviews-2023) | 1996~2023년 리뷰 + 상품 메타데이터(가격, 카테고리, 평점) 수억 건 | 무료, Hugging Face. 카테고리 2~3개만 샘플링해서 사용 |
| [Keepa API](https://keepa.com/#!api) | 가격·판매랭킹(BSR) 히스토리 | 유료(월 €19~). 핵심 ASIN 수백 개만 1개월 구독으로 수집 가능. 예산 없으면 생략 가능 |
| 환율·물류비 등 보조 데이터 | 마진 시뮬레이션용 | 공공데이터/수기 가정 |

#### 분석 설계 (예시)
1. **카테고리 경쟁 구조 EDA** — 카테고리별 리뷰 수 분포(상위 셀러 집중도), 평점 분포, 가격대 분포로 "레드오션 지수" 정의
2. **니치 기회 스코어링** — 수요 프록시(리뷰 증가율) 대비 공급 프록시(경쟁 상품 수·신규 진입 속도)로 진입 매력도 점수화. *본인이 실제 팔았던 카테고리를 사례로 검증 → 포트폴리오의 백미*
3. **리뷰 텍스트 마이닝** — 저평점 리뷰에서 미충족 니즈(unmet needs) 추출 (예: "사이즈가 작다"가 반복되는 니치 = 개선 상품으로 진입 기회). LLM/NLP 활용 지점
4. **가격 포지셔닝 분석** — 가격대별 평점·리뷰 획득 속도 관계, 신규 진입자의 최적 가격대 제안
5. **머신러닝** — "출시 후 1년 내 리뷰 N개 도달(성공)" 예측 모델 → 어떤 초기 조건(가격, 카테고리, 첫 달 평점)이 성공을 가르는지 피처 중요도 해석
6. **최종 산출물** — "신규 셀러를 위한 카테고리 진입 가이드" 형태의 의사결정 리포트 + 대시보드(Streamlit/Tableau)

#### 차별화 포인트 정리
- 문제 정의가 본인 사업 경험에서 나옴 (면접 스토리텔링 최강)
- 구매자 관점이 아닌 **공급자/전략 관점** → 전략기획 직무까지 커버
- "성공 예측 모델"은 단순 분류 문제가 아니라 비즈니스 의사결정(진입/비진입)에 연결됨

#### 리스크/주의
- Amazon Reviews 2023은 용량이 큼 → 반드시 2~3개 카테고리로 범위 한정
- "리뷰 감성분석 했습니다"로 끝나면 흔한 프로젝트로 전락 → 모든 분석이 **"진입할까 말까"라는 하나의 의사결정 질문**에 수렴하도록 유지

## 3. 차선 후보 (비교용)

### 후보 B: "투자 대회 수상자의 셀프 복기 — 개인투자자 행동 편향 분석" (핀테크)
- **컨셉**: 본인 대회 전략을 회고하며, 공개 시장 데이터(KRX, 한국예탁결제원 [공공데이터포털 API](https://www.data.go.kr/data/15001145/openapi.do), yfinance)로 개인투자자의 행동 편향(처분효과, 테마 쏠림 등)을 정량 분석. 증권사/토스증권류 핀테크의 "투자자 보호·리텐션" 문제와 직결
- **장점**: 수상 이력과 시너지, 핀테크 도메인 직격
- **약점**: 본인 매매 기록이 없어 서사가 대표 프로젝트보다 한 단계 약함. 시장 데이터 분석은 경쟁자도 많은 편

### 후보 C: "이커머스 결제 이상거래(FDS) 탐지" (이커머스 × 핀테크 교집합)
- **컨셉**: 불균형 데이터 처리 + 비용 기반 평가(오탐의 비즈니스 비용)를 중심으로 한 사기 탐지 모델
- **장점**: 핀테크 면접 단골 주제, ML 깊이 어필
- **약점**: 공개 데이터셋(IEEE-CIS 등)이 매우 흔해 차별화가 어려움 — 본인 경험과의 연결고리도 약함. **단독 대표 프로젝트로는 비추천**, 추후 2번째 프로젝트로 적합

### 추천 순위
**A(아마존 셀러 진입 분석) ≫ B(투자자 행동 분석) > C(FDS)**

A를 대표 프로젝트로 완성한 뒤 여력이 생기면 B를 가볍게 추가하는 것이 이커머스·핀테크 양쪽 지원에 최적 조합입니다.

## 4. 이력서/면접 연결 팁

- 프로젝트 제목에 "전직 아마존 셀러" 정체성을 드러낼 것. 예: *"3년차 셀러가 데이터로 다시 푸는 카테고리 진입 전략"*
- README/리포트 도입부에 실제 셀러 시절 의사결정 일화 1개를 배치 → 채용담당자의 체류 시간을 늘리는 훅
- 반도체 영업 경력은 이력서에서 빼더라도, 면접에서 "고객사 데이터 기반 커뮤니케이션" 질문이 오면 활용 가능하니 답변은 준비해둘 것
- 모든 차트·모델에 "그래서 무엇을 결정해야 하는가(So what)"를 한 줄씩 달 것 — 2026년 채용 트렌드는 기술 스택보다 판단력·커뮤니케이션 검증에 무게
